# Правила проекта
Universal Rules for High-Load Bot Development
1. Scalability & Performance First
Проектируй код с расчетом на 100k+ активных пользователей. Избегай решений, которые создают «бутылочное горлышко» в базе данных или памяти.
Используй асинхронность как стандарт. Любая тяжелая операция (загрузка медиа, сложные анимации) должна выполняться в фоновом режиме, не блокируя основной цикл обработки сообщений.
2. Modern Tech Stack (Node.js/TypeScript)
Пиши на TypeScript с использованием строгой типизации. Это поможет избежать ошибок при масштабировании логики.
Используй фреймворк grammY как основной инструмент для работы с Telegram API из-за его высокой производительности и модульности.
3. State & Cache Management
Разделяй данные: PostgreSQL (Supabase) — для долгосрочного хранения (юзеры, оплаты), Redis — для горячих данных (кэш сообщений для Spy-функций, временные состояния, лимиты).
Для Spy-логики реализуй эффективное кэширование входящего потока сообщений, чтобы мгновенно сравнивать версии при редактировании/удалении.
4. Resilience (Отказоустойчивость)
Внедряй Graceful Degradation: если одна функция (например, анимация) падает из-за лимитов Telegram, основной бот и Spy-модуль должны продолжать работать.
Обязательная обработка ошибок Telegram API (особенно FloodWait) с использованием очередей или механизмов повтора (retry).
5. Clean Code & Modularity
Соблюдай принцип Single Responsibility: логика анимаций, логика шпиона и логика оплаты должны быть в разных модулях.
При создании UI (Inline-меню, Mini App) придерживайся минимализма и нативности Telegram.
